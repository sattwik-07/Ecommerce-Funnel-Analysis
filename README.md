#E-commerce Conversion Funnel Analysis

A full end-to-end data analysis project that tracks a synthetic e-commerce
store's user journey — **Visit → Product View → Add to Cart → Checkout →
Purchase** — to find where customers drop off, quantify the impact, and
recommend fixes. Built with a Python analysis pipeline and a self-contained,
interactive live dashboard.

**[Live Dashboard Demo](#-interactive-dashboard)** · [Key Findings](#-key-findings) · [Recommendations](#-recommendations) · [How to Run](#️-how-to-run)

## 📌 About This Project

This started as a hands-on exercise in the kind of funnel analysis product
and growth teams do every day: given raw event data, figure out *where*
users are being lost and *why*, then turn that into concrete
recommendations. Rather than stop at a static notebook, I built it as a
complete pipeline (data → analysis → charts) plus a dashboard people can
actually click through — including uploading their own dataset.

**What it demonstrates:**
- Funnel / cohort-style analysis with pandas
- Turning raw event logs into segment-level conversion metrics
- Data visualization (matplotlib for static reports, hand-rolled canvas
  charts for the live dashboard)
- Building a dependency-free, single-file interactive web dashboard in
  vanilla JS (no frameworks, no build tools)
- Client-side CSV parsing and on-the-fly re-analysis of arbitrary user data

## 🧭 Methodology

1. **Funnel definition** — five ordered stages: `visit → product_view →
   add_to_cart → checkout → purchase`. A session's funnel progress is
   defined as the *furthest* stage it reached.
2. **Conversion rate** between two consecutive stages = sessions reaching
   stage *N+1* ÷ sessions reaching stage *N*.
3. **Drop-off rate** at a stage = `100% − conversion rate` into that stage.
4. **Bottleneck** = the stage transition with the single highest drop-off
   rate across the whole funnel.
5. **Segmentation** — the same funnel math is recomputed independently for
   `device`, `traffic_source`, `product_category`, and `country` to see
   where performance diverges from the overall average.
6. **Revenue** — computed only on sessions that reached `purchase`, using a
   category-level average order value with realistic noise.

## 📊 Key Findings

**Dataset:** 50,000 sessions, Jan–Mar 2025 (synthetic, generated by
`src/generate_data.py`)

### Overall Funnel

| Stage | Sessions | % of Visitors | Conversion into stage | Drop-off |
|---|---:|---:|---:|---:|
| Visit | 50,000 | 100.0% | — | — |
| Product View | 29,486 | 58.97% | 58.97% | 41.03% |
| Add to Cart | 10,336 | 20.67% | 35.05% | 64.95% |
| Checkout | 4,396 | 8.79% | 42.53% | 57.47% |
| Purchase | 2,998 | **6.00%** | 68.20% | 31.80% |

**Overall Visit → Purchase conversion rate: 6.00%**

### 🚨 Bottleneck: Add to Cart → Checkout

This is the single largest leak in the funnel — **64.95% of users who added
a product to their cart never started checkout** (19,150 sessions lost).
By contrast, once someone *does* start checkout, 68.2% go on to purchase —
the funnel's healthiest transition. This tells us the problem isn't buyer
intent late in the journey; it's friction and hesitation right after the
cart step.

### Segment Insights

- **Device** — Desktop converts end-to-end at **8.40%**, versus **4.53%**
  on Mobile and 6.00% on Tablet. The gap is concentrated in the cart →
  checkout step specifically: Desktop converts that transition at **54.7%**
  vs only **34.96%** on Mobile — a strong signal of mobile checkout friction
  (payment entry, form length, page load).
- **Traffic Source** — Email-sourced sessions convert best overall
  (**8.06%**), followed by Organic Search (**6.97%**). Paid Ads convert
  worst (**4.33%**), despite being the second-largest traffic source —
  suggesting lower purchase intent or an ad/landing-page mismatch.
- **Product Category** — Fashion (6.57%) and Sports (6.05%) convert best;
  Books convert lowest (5.56%), consistent with a lower-consideration,
  lower-AOV category.
- **Revenue** — ₹67.97L in total revenue across 2,998 orders, average order
  value ≈ ₹2,267.

## 💡 Recommendations

1. **Simplify the Add-to-Cart → Checkout step.** Add a persistent mini-cart
   with a one-click "Proceed to Checkout" CTA immediately after an add,
   instead of requiring users to navigate back to the cart page.
2. **Reduce mobile payment friction.** Prioritize saved payment methods,
   autofill, and one-tap wallets (UPI / Apple Pay / Google Pay) specifically
   on mobile checkout, where the cart → checkout conversion gap is largest.
3. **Re-align Paid Ads creative with landing pages.** Paid traffic views
   fewer products per session than organic traffic — tightening ad-to-
   landing-page relevance should lift product-view engagement.
4. **Launch an abandoned-checkout recovery flow.** Checkout already
   converts well once started (68.2%); a 1-hour and 24-hour email/SMS
   nudge to abandoned checkouts can recover a meaningful share of
   near-converters.
5. **Invest further in Email as a channel.** It already has the best
   end-to-end conversion rate — expanding send frequency to high-intent
   segments (past cart-adders, past purchasers) is likely to compound.

## 🔁 Using This on Your Own Data

Two ways to analyze your own dataset instead of the bundled sample:

**A. Via the dashboard (no code):** open `dashboard/index.html`, go to
**Data Source**, and upload a CSV with columns `session_id, device,
traffic_source, country, product_category, stage, event_timestamp,
order_value (optional)`, where `stage` uses the funnel labels `visit`,
`product_view`, `add_to_cart`, `checkout`, `purchase`.

**B. Via the Python pipeline:** replace `data/ecommerce_events.csv` with
your own export using the same schema, then re-run
`python src/funnel_analysis.py`, `python src/make_charts.py`, and
`python src/build_dashboard.py`.
